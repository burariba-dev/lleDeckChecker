ラブカデッキチェック＆使用率チェッカー
- バージョン情報
  - 2026/2/24 v1.0.0 公開
    - 通常版 約 70MB 追加インストール不要
    - 軽量版 約100KB .Net 8系の動作環境が必要
- 機能
  - デッキコードからデッキリストを自動取得
    （開発途中の為、ラブカのリストかどうかを判別する機能はありません）
  - 複数のデッキから全採用枚数を確認し、使用率選抜による採点を実施可能
- 使い方
  - 参加者及びデッキの登録
    - プレイヤー追加ボタンから追加
      1. プレイヤー名とデッキコードを入力後「デッキコード読み込み」ボタンを押下
      2. 正常なデッキコードの場合は1~2秒でリストが表示される
      3. 「登録」「キャンセル」で確定 or 取り消しが可能
    - ファイルからまとめて追加
      1. 1行が「プレイヤー名,デッキコード」のcsv形式になったcsvもしくはtxtファイルを用意する
      2. ボタン押下でファイル選択ダイアログが表示されるので１で作ったファイルを選択
      3. ファイルの読み込み開始。ユーザー数×2秒程度の時間がかかる
      4. プレイヤー名とデッキコードがまとめて表示されれば完了
      5. デッキコードが正しくない等の場合該当プレイヤーは登録されないので、総数があっているかを確認してください
  - 使用率の計算
    - 全数計算
      1. そのまま「使用率計算」ボタンを押下
      2. 「計算が終了しました。」のメッセージを待つ（すぐ終わる）
      3. プレイヤーリストの「合計使用率点数」の欄が埋まる（筈）
    - 勝ち数計算（部分計算）
      1. プレイヤーの横にある「勝利数」の欄を手動で書き換える（お手数おかけします）
      2. 計算ボタンの左にある「0」勝以上の欄を絞り込みたい勝数にセット
      3. 「使用率計算」ボタンを押下（以下は全数計算と同様）
    - 出力部分
      - 実行ファイル（exe）と同階層に「Decks」と「Datas」フォルダを作成
      - Decksには各プレイヤーのデッキリストと点数などを書いたcsvを出力する
      - Datasには集計対象の全カードと使用枚数のランキングを書いたcsvを出力する
      - メモ帳、Excel、Googleスプレッドシートなどで表示可能
      - 出力のたびに上書きするので、データとして取っておく場合は注意
